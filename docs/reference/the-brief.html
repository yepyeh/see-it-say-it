The Brief

Development Brief: "See It Say It" (PWA)

1. Project Overview

See It Say It is a public-facing platform for reporting community issues (potholes, fly-tipping, graffiti, etc.). The goal is to create a seamless, visual reporting flow that automatically routes data to the correct local authorities.

2. Core Technology Stack

  • Framework: Astro (SSR Mode).
  • Platform: Cloudflare (Pages, Workers, D1 Database, R2 Storage).
  • PWA (Progressive Web App): We have chosen the PWA route for cross-platform compatibility (iOS/Android) without the overhead of app stores, allowing for offline caching and "Add to Home Screen" functionality.
    • Note to Developer: We are open to Capacitor if you believe a native wrapper is necessary for better GPS/Camera background stability, but PWA is the current preference.
  • Authentication: OTP (One-Time Password) via Email or SSO (Google/Apple). No legacy passwords.

3. The Reporting Engine (The "Multi-Step" UI)

Reporting must be visual and "frictionless." The UI should follow a state-machine pattern:
  1. Capture: Upload/Take photo (Store in Cloudflare R2).
  2. Locate: Auto-detect GPS; provide a map interface for manual pinning.
  3. Categorize: A "Bento Box" or grid of visual icons.
  4. Detail: Short text description and "Severity" toggle.
  5. Submit: Summary screen with "Optimistic UI" updates.

4. Technical Requirements & Database Schema

The foundation must use CSS variables and a clean database structure to allow for theme toggling (Light, Dark, Comfy, Medium-Dark).

Proposed Database (D1) Schema:

FieldTypeDescription
report_idUUIDPrimary Key
user_idUUIDForeign Key to Users table
lat_longPoint/FloatPrecise coordinates for mapping
categoryStringe.g., "Fly-tipping", "Road Damage"
statusEnumDraft, Submitted, Dispatched, In-Progress, Resolved
media_urlsArrayLinks to R2 buckets
authority_idStringID of the routed agency
severityInteger1-5 scale

5. Automation & Routing Logic

We want the system to be as automated as possible.
  • Geo-Routing: The app should use Geo-fencing logic. When a report is submitted, a Cloudflare Worker should check the coordinates against a GeoJSON map of local council boundaries.
  • Dispatch: Once the boundary is identified, the system triggers an automated email or API call to the relevant department (e.g., Highways, Environment).
  • Spam Prevention: Integrate Cloudflare Turnstile to ensure reports are genuine without hindering the user experience.

6. User Experience (UX) Goals

  • Themeable Foundations: Use a library like Shadcn/ui or DaisyUI.
  • Density Settings: Users should be able to switch between "Compact" (data-heavy) and "Comfy" (large tap targets, more white space).
  • Public Map: A main dashboard showing a filtered view of all active reports in the user’s vicinity.

7. User Stories (For Testing)

  • As a Resident, I want to take a photo of a pothole and have my location automatically tagged so I don't have to type an address.
  • As a User, I want to receive a notification when the status of my report changes from "Submitted" to "In Progress" so I know I've been heard.
  • As a Community Member, I want to see a map of my local area so I can see if an issue has already been reported by someone else.

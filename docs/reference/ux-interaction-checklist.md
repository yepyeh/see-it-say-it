# **UX Interaction Checklist: The Reporting Drawer**

When reviewing the next "Tier 1 → Tier 2" implementation, check for these "Premium Utility" sensations:

### **1\. The "Vaul" Feel (Mobile)**

* \[ \] **Rubber Banding:** When you pull the drawer down, does it resist slightly before snapping shut? (This is a hallmark of high-quality PWAs).  
* \[ \] **Snap Points:** Does the drawer have a "Half-height" state (to see the map) and a "Full-height" state (for selecting sub-categories)?

### **2\. The Taxonomy Transition**

* \[ \] **No Page Loads:** The switch from "Roads" (Tier 1\) to "Potholes" (Tier 2\) should feel like a horizontal slide or a quick fade—never a browser refresh.  
* \[ \] **Breadcrumbs:** Is there a clear "Back" button or a small label (e.g., "Roads \>") so the user doesn't feel lost in the second tier?

### **3\. Spatial Awareness**

* \[ \] **Background Dimming:** Does the map darken slightly (20-30%) when the drawer is active to draw focus to the selection?  
* \[ \] **The "Pin" Anchor:** Does the report location pin stay dead-center in the visible portion of the map?

### **4\. Search Velocity**

* \[ \] **Instant Results:** If you use the search bar, do the categories filter instantly as you type?  
* \[ \] **Keyboard Interaction:** Does the mobile keyboard push the drawer up correctly so the search input isn't hidden?
